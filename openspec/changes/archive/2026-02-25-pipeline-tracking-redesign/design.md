## Context

`renderPipelineTracking` 当前将所有分组内容渲染进一个大表格，分组只靠表头区分。岗位库页面用卡片 section 展示，视觉层次更清晰。本次只改前端渲染，不动 API。

## Goals / Non-Goals

**Goals:**
- 每个分组渲染为独立卡片（与岗位库风格一致）
- 卡片头部：分组名 + 人数 badge，按岗位分组时名称可点击跳转看板
- 卡片内部：表格展示姓名、阶段（按岗位分组时）或岗位（按阶段分组时）、最后更新
- 保留分组切换下拉和岗位筛选下拉

**Non-Goals:**
- 不改动 API 或数据结构
- 不新增排序功能
- 不改动卡片内的操作按钮（当前无操作按钮，保持现状）

## Decisions

**复用现有卡片 CSS 类**：直接用 `.card` + `padding:0;overflow:hidden` 风格，与岗位库保持一致，无需新增样式。

## Risks / Trade-offs

- 无显著风险，纯前端渲染逻辑调整
